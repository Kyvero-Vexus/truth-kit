# tools

This directory holds concrete tools, adapters, and reusable libraries.

Serious capabilities in `truth-kit` should ideally exist here before being
wrapped by workflows or agents.

Tools should be usable directly whenever practical, such as through:

- CLI entry points
- library APIs
- batch or benchmark modes

Subdirectories:

- `archive/`
- `reverse-search/`
- `provenance/`
- `forensics/`
- `benchmarks/`
- `common/`
